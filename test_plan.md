# DEU 测试计划

## 1. 测试策略概述

### 1.1 测试层次

```
┌─────────────────────────────────────────┐
│  L3: 顶层集成测试  deu_design            │  覆盖完整流水线、时序、边界
├─────────────────────────────────────────┤
│  L2: 模块级测试   bitonic_sort           │  覆盖排序正确性、流水延迟
├─────────────────────────────────────────┤
│  L1: 单元测试     sliv_decoder           │  覆盖所有 SLIV 分支
│                   decomp_square          │  覆盖所有 exp 值和符号
│                   compare_swap           │  覆盖升降序、等值 tie-breaker
└─────────────────────────────────────────┘
```

### 1.2 验证方法

- **参考模型**：Python 参考实现，与 RTL 输出逐拍比对
- **定向测试**：覆盖边界值、特殊情况
- **随机测试**：随机生成 SLIV + 数据，批量验证
- **波形检查**：关键信号时序人工确认

### 1.3 通过准则

| 级别 | 标准 |
|------|------|
| 单元 | 所有定向用例通过；随机 10000 拍无错误 |
| 模块 | 所有定向用例通过；随机 50000 拍无错误 |
| 集成 | 所有定向用例通过；随机 100000 拍无错误；流水延迟精确验证 |

### 2.2 decomp_square

**测试要点：** exp=0/exp>0 分支、符号位忽略、val16 前缀、移位量、最大/最小值

| 用例编号 | exp | sign | data | 期望 decp_tmp | 期望 square | 说明 |
|---------|-----|------|------|--------------|------------|------|
| DS-01 | 0  | 0 | 0       | 0           | 0                    | 全零 |
| DS-02 | 0  | 0 | 1       | 1           | 1                    | exp=0 最小非零 |
| DS-03 | 0  | 0 | 0x7FFF  | 32767       | `32767²`             | exp=0 最大 |
| DS-04 | 0  | 1 | 0x7FFF  | -32767      | `32767²`             | 负数平方=正数平方 |
| DS-05 | 1  | 0 | 0       | 32768       | `32768²`             | exp=1 data=0，shift=0 |
| DS-06 | 1  | 0 | 0x7FFF  | 65535       | `65535²`             | exp=1 最大 |
| DS-07 | 2  | 0 | 0       | 65536       | `65536²`             | exp=2 data=0，shift=2 |
| DS-08 | 2  | 0 | 0x7FFF  | 131071      | `131071²`            | exp=2 最大 |

**随机测试：** 随机生成 exp(0~15)、sign(0/1)、data(0~32767)，与参考模型比对。

### 2.3 compare_swap

**测试要点：** 升降序、相等值 tie-breaker、不交换/需交换的边界

| 用例编号 | din_a | idx_a | din_b | idx_b | dir | 期望 lo(data,idx) | 期望 hi(data,idx) | 说明 |
|---------|-------|-------|-------|-------|-----|-----------------|-----------------|------|
| CS-01 | 10 | 3 | 20 | 1 | 0(asc) | (10,3) | (20,1) | a<b，不交换 |
| CS-02 | 20 | 3 | 10 | 1 | 0(asc) | (10,1) | (20,3) | a>b，交换 |
| CS-03 | 10 | 3 | 20 | 1 | 1(desc) | (20,1) | (10,3) | desc，a<b 交换 |
| CS-04 | 20 | 3 | 10 | 1 | 1(desc) | (20,3) | (10,1) | desc，a>b 不交换 |

| CS-09 | `64'hFFFF...` | 0 | `64'hFFFF...` | 15 | 0 | (MAX,0) | (MAX,15) | 同为 MAX，idx 排序 |

## 3. 模块级测试：bitonic_sort

### 3.1 功能正确性

| 用例编号 | 输入描述 | 验证要点 |
|---------|---------|---------|
| BS-01 | 16 路随机不重复值，全有效(bitmap=0xFFFF) | 输出严格升序，索引对应正确 |
| BS-02 | 16 路全相同值，全有效 | 输出升序但 idx 也从小到大（0~15）|
| BS-03 | 部分相同值（4种值各4个），全有效 | 等值按 idx 排序 |
| BS-04 | 8 路有效(bitmap=0x3366, S=2,L=8) + 8路 MAX | 有效值排在低位，MAX 排在高位；o_dout_vld 低8位为1 |
| BS-05 | 1 路有效(bitmap=0x0001) + 15路 MAX | o_dout_vld=16'h0001，o_dout0=有效值 |
| BS-06 | 全部无效(bitmap=0x0000，所有输入=MAX) | o_dout_vld=16'h0000 |
| BS-07 | 全部有效(bitmap=0xFFFF) | o_dout_vld=16'hFFFF |
| BS-08 | 已升序输入 | 输出与输入相同（不改变顺序） |
| BS-09 | 已降序输入 | 输出完全翻转 |
| BS-10 | 随机顺序，已知答案 | 逐元素比对 |

### 3.2 流水线时序

| 用例编号 | 验证要点 |
|---------|---------|
| BS-T01 | i_vld=1 输入后，第 3 个上升沿 o_dout_vld 变有效（3拍流水延迟） |
| BS-T02 | 连续 N 拍有效输入，输出连续 N 拍有效，无气泡 |
| BS-T03 | i_vld=0 期间，o_dout_vld=0；i_vld 重新拉高后正确输出 |
| BS-T04 | 复位后 o_dout_vld=0，rst_n 释放后数据路径干净 |

### 3.3 数据保持验证（功耗优化正确性）

| 用例编号 | 验证要点 |
|---------|---------|
| BS-P01 | i_vld=0 时，ra_d*/ra_i*/ra_bitmap 不变（无翻转） |
| BS-P02 | ra_vld=0 时，rb_d*/rb_i*/rb_bitmap 不变 |
| BS-P03 | rb_vld=0 时，o_dout*/o_idx* 不变 |

## 5. 参考模型设计

### 5.1 Python 参考模型接口

```python
def deu_ref_model(sliv, cmp_data_list):
    """
    输入：

def decomp_square(cmp_data): ...

# Step3: 带 MAX 填充后排序（含 tie-breaker）
def sort_with_mask(squares, bitmap): ...
```

### 5.3 比对逻辑

每个有效输出拍（o_dout_vld 非零时），比对：
- `o_dout_vld` 是否等于参考的 `(1 << K) - 1`（K=popcount(bitmap)）
- `o_dout[k]` 是否等于参考排序第 k 个的 square 值（仅 o_dout_vld[k]=1 时）
- `o_dout[k]_idx` 是否等于参考排序第 k 个的原始索引

## 6. 测试优先级与执行顺序

```
P0（必须通过）:
  1. SD-03（README 标准示例）
  2. DT-01（顶层标准用例）
  3. DT-T01（流水延迟精确验证）
  4. DT-T02（连续输入）

P1（高优先级）:
  5. SD-01~SD-10（全 SLIV 分支）
  6. DS-01~DS-12（全解压缩边界）
  7. CS-01~CS-09（compare_swap 全场景）
  8. BS-01~BS-10（排序正确性）
  9. DT-D01~DT-D07（数据边界）
  10. DT-S01~DT-S06（SLIV 覆盖）

P2（回归）:
  11. DT-R01~DT-R03（随机回归）
  12. BS-P01~BS-P03（功耗寄存器保持）
```

## 7. 关键波形检查点

基于时序图中的节点标注（`a~b`, `e~b 0-40 cycle`）：

| 检查项 | 描述 |
|-------|------|
| C1 | `i_data_vld` 上升沿（节点 a/e）→ `o_dout_vld` 上升沿（节点 b）延迟 = 6 拍 |
| C2 | 复位期间 `o_dout_vld = 16'h0000` |
| C3 | `i_data_vld=0` 期间，Stage1~3 的 r*_vld 均在对应延迟后为 0 |
| C4 | 数据寄存器在 vld=0 期间无跳变（功耗验证） |
| C5 | 等值情况下低索引数据出现在低编号输出端口 |
